## 📌 Executive Summary
This repository documents a structured, intensive progression through RESTful API engineering using Python. It serves as a living codebase demonstrating the evolution from consuming third-party data to architecting and deploying secure, production-ready server applications.

## 🏗️ The Curriculum Framework

### Phase 1: Data Consumption & Protocol Mastery (Completed)
* **HTTP Protocol:** Deep understanding of methods (GET, POST, PUT, DELETE) and status codes.
* **Data Serialization:** Parsing and manipulating complex nested JSON payloads.
* **Query Operations:** Dynamically interacting with API endpoints via URL parameters.

### Phase 2: Application Building & Security (Completed)
* **Authentication:** Secure handling of API Keys using environment variables (`.env`).
* **Version Control:** Implementation of `.gitignore` best practices to prevent credential leakage.
* **Real-World Implementation:** Built [Atmos-Live-API / Nimbus], a live weather data aggregation tool.

### Phase 3: Server Architecture (In Progress...)
* **Frameworks:** Transitioning to **FastAPI** to build high-performance, asynchronous REST APIs.
* **Routing & Endpoints:** Designing scalable URI structures.

## 🛠️ Technology Stack
* **Language:** Python 3.x
* **Libraries:** `requests`, `python-dotenv`, `FastAPI` (upcoming)
* **Concepts:** REST Architecture, JSON Data Modeling, Secure Credential Management
